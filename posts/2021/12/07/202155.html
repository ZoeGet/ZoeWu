Linux常用命令

Created2021-12-07|Updated2021-12-07
|Post Views:

pwd

查看当前的工作目录

[root@localhost yum.repos.d]# cd /var/mail

[root@localhost mail]# pwd

/var/mail

cd

切换工作目录

ls

列出目录及文件名列表显示目录内容

-a：全部的文件，连同隐藏文件（开头为 . 的文件）一起列出

-d：仅列出目录本身，而不是列出目录内的文件数据

-l：长数据串列出，包含文件的属性与权限等等数据

将家目录下的所有文件列出来（含属性与隐藏档）

ls -al /root

du

统计目录及文件所占用的磁盘空间

-a或–all：显示目录中文件的大小

-h或–human-readable：以K、M、G为单位，提高信息的可读性

-s或–aummarize：仅显示总计

例：

[root@localhost mail]# cd /boot/

[root@localhost boot]# du -sh

118M .

[root@localhost boot]# du -ah

0 ./efi/EFI/redhat

0 ./efi/EFI

0 ./efi

4.0K ./grub2/device.map

8.0K ./grub2/i386-pc/gcry_md5.mod

16K ./grub2/i386-pc/acpi.mod

4.0K ./grub2/i386-pc/gcry_rfc2268.mod

mkdir

创建新的目录

-p 选项：递归创建所有目录，以创建/root/test/demo/lmS/mkdir为例，在默认情况下，你需要一层一层创建各个目录，而使用-p选项，则系统会自动帮你创建/test、/test/demo以及/test/demo/lmS和/test/demo/lmS/mkdir

touch

创建空文件更新时间戳

例1：当指定操作文件不存在时，该命令会在当前位置建立一个空文件

[root@localhost ~]# touch bols

[root@localhost ~]# ls

anaconda-ks.cfg bols

例2：当文件存在时，会修改此文件的时间参数（时间戳）

[root@localhost ~]# ls -l

总用量 4

-rw——-. 1 root root 1770 9月 3 23:02 anaconda-ks.cfg

-rw-r–r–. 1 root root 0 12月 7 21:28 bols

[root@localhost ~]# touch bols

[root@localhost ~]# ls -l

总用量 4

-rw——-. 1 root root 1770 9月 3 23:02 anaconda-ks.cfg

-rw-r–r–. 1 root root 0 12月 7 21:30 bols

ln

创建链接 link

软链接：当删除原文件后，软连接就不能打开（相当于将原文件创建一个快捷方式到相应目录下） 选项：-s

硬链接：当删除原文件后，硬链接仍可以打开（复制一个文件到相应目录下）

例：

[root@localhost ~]# touch f1 #创建一个测试文件f1

[root@localhost ~]# ln f1 f2 #创建f1的一个硬链接文件f2

[root@localhost ~]# ln -s f1 f3 #创建f1的一个软连接文件f3

cp

复制文件或目录

-i：覆盖目标同名目录或文件时

-p：复制时保留原文件的权限、时间标记等属性不变

-r：复制目录时必须使用此选项，表示递归复制所有文件及子目录

rm

删除文件或目录

-f：删除文件或目录时不进行提醒，可直接强制删除

-i：删除文件或目录时提醒用户确认

-r：删除目录时必须使用此选项，表示递归删除整个目录树即包括所有的子目录和文件

（如果任何选项都不加，则默认执行的是rm -I ，也就是删除一个文件之前会先询问是否删除）

mv

移动文件或目录

格式：mv [选项] 源文件 目标文件

-f：强制覆盖，如果目标文件已经存在，则不询问，直接强制覆盖

-i：交互移动，如果目标文件已经存在，则询问用户是否覆盖（默认选项）

-n：如果目标文件已经存在，则不会覆盖移动，而且不询问用户

find

可以根据目标的名称、类型和大小等不同属性进行精细查找

find命令在查找时采用递归的方式，其使用形式相当灵活

格式：find [查找范围] [查找条件表达式]

mount

挂载Linux系统外的文件，它可以将分区挂接到Linux的一个文件下

格式：mount [-t 文件系统] 设备 挂载点
