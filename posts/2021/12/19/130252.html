远程控制

Created2021-12-19|Updated2021-12-19
|Post Views:

安装软件包：yum -y install openssh

控制服务sshd.service

启动、停止或重新启动sshd.service

查看sshd状态：[root@localhost ~]# systemctl status sshd

停止sshd当前运行： [root@localhost ~]# systemctl stop sshd

禁用sshd运行： [root@localhost ~]# systemctl disable sshd

重新启动sshd： [root@localhost ~]# systemctl restart sshd

启动sshd： [root@localhost ~]# systemctl enable sshd

启用sshd.service

查看sshd.service服务状态

查看sshd的端口侦听信息

禁止超级用户远程登陆

（1）若想禁止超级用户root使用SSH登录，则可以将/etc/ssh/sshd_config配置文件中的“PermitRootLogin”参数值设置为no

（2）若想禁止超级用户root基于密码的身份验证，但是允许对root执行基于密钥的身份验证，可将“PermitRootLogin”参数值设置为without-password

注：配置完该主配置文件后，必须通过“systemctl reload sshd.service”命令重新加载sshd服务才能使更改生效。

ssh：ssh命令用于登录远程主机，并且在远程主机上执行命令

[root@localhost ~]# ssh root@192.168.47.30

sftp：sftp可以用来打开一次安全互动的FTP会话，它与ftp相似，只不过使用安全且加密的连接 lls：查看本地目录内容 ls：查看远程服务器目录内容

[root@localhost ~]# sftp 192.168.47.30

使用 get 命令可以从远程服务器拉取文件到本地：

sftp> get remoteFile [newName]

sftp> get file1 /tmp

如果不指定 newName，将使用和远程服务器相同的文件名。

使用 -r 选项可以拉取整个目录：

sftp> get -r fengmengyu/ /tmp/feng

sftp> get -r fengmengyu

使用 put 命令可以从本地上传文件到服务器：

sftp> put localFile

同样的，可以使用 -r 参数来上传整个目录，但是有一点要注意，如果服务器上不存在这个目录需要首先新建：

sftp> mkdir folder

sftp> put -r folder/

scp：scp的作用是将文件安全的复制到远程主机或本地主机上，这取决于所要发送的文件的位置，必须指定用户名、主机名、目录和文件。

复制文件

[root@localhost ~]# scp file1 root@192.168.47.30:/root

复制文件夹

[root@localhost ~]# scp -r localfolder/ root@192.168.47.30:/root
