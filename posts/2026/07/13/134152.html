修复结果

此后，在终端中执行：

会优先使用 Visual Studio Code 打开当前目录。其他依赖 code 命令的脚本和开发工具也会优先调用 VS Code。

关于 Codex 中没有 VS Code 选项的问题

虽然 Windows 默认的 code 命令已经切换为 Visual Studio Code，但 ChatGPT Desktop（Codex）设置中的“默认文件打开目标”仍然可能只有：

  • Cursor
  • Default app
  • File Explorer

没有 Visual Studio Code 选项。

这是因为该下拉列表由 ChatGPT Desktop 客户端当前支持的 IDE 类型决定，并不是根据 Windows 的 PATH 环境变量，也不是根据 code 命令当前指向的程序动态生成。

因此，调整环境变量可以解决终端和相关工具优先启动 Cursor 的问题，但不会让 ChatGPT Desktop 的下拉列表自动增加 Visual Studio Code 选项。

如果当前版本没有提供 VS Code 选项，可以暂时选择 Default app 或 File Explorer，并等待后续客户端版本增加对 Visual Studio Code 的直接支持。

总结

这次问题实际上包含两个不同的部分：

  1. Windows 中的 code 命令被 Cursor 的路径优先接管；
  2. ChatGPT Desktop（Codex）的默认文件打开目标中没有提供 VS Code 选项。

但 Codex 设置中的 IDE 选项由客户端自身决定。如果当前版本不支持 Visual Studio Code，修改 PATH 后也不会自动出现该选项，只能等待后续版本提供支持。
