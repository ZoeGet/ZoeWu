问题现象

在执行 Hexo 部署命令时，可能会遇到下面的 Git 报错：

这个报错通常会导致 hexo d 部署失败，无法正常把生成后的静态文件推送到远程仓库。

问题原因

Git 在执行 Hexo 部署时，检测到 .deploy_git 仓库的所有者，也就是 Owner，与当前登录的 Windows 用户不一致。

为了保证仓库安全，Git 默认会禁止当前用户继续操作该仓库，因此 Hexo 部署时会失败，并提示：

这个问题与 Hexo 本身关系不大，主要是 Git 的安全检查机制导致的。

问题产生原因

该问题通常出现在以下几种情况：

  • Windows 系统重装或更换了登录账户；
  • .deploy_git 文件夹从其他电脑复制而来；
  • 曾使用管理员身份运行 Git，之后又使用普通用户运行；
  • 仓库文件的所有者，也就是 Owner，发生了变化。

Git 2.35 及以上版本新增了仓库安全检查机制。当 Git 发现当前用户与仓库所有者不一致时，会认为该仓库可能存在安全风险，因此拒绝继续操作。

建议

优先采用方法一，将 .deploy_git 添加到 safe.directory，这是更推荐的处理方式。

如果添加安全目录后仍然无法部署，再删除 .deploy_git 文件夹，让 Hexo 重新生成部署仓库即可。
