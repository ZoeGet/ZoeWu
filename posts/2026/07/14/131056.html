前言

因此，可以使用 Conventional Commits 规范来统一 Git 提交信息。

什么是 Conventional Commits

Conventional Commits 可以翻译为“约定式提交”或“规范化提交”。

它是一套 Git 提交信息书写规范，用来让提交记录具备清晰、统一、可读、可解析的结构。

scope 是可选项，用来说明这次提交影响的范围。

常见的 scope 可以是：

  • 模块名：led、display、api、auth；
  • 文件或文档名：readme、config、gitignore；
  • 功能区域：login、deploy、driver；
  • 平台或工具：windows、hexo、github-actions。

如果项目比较小，也可以不写 scope。

Description 简短描述

description 是提交信息中最重要的一部分，用来简短说明这次提交做了什么。

建议：

  • 使用简短明确的描述；
  • 使用英文小写开头；
  • 不要以句号结尾；
  • 尽量使用动词开头；
  • 描述“做了什么”，而不是“我做了什么”。

总结

Conventional Commits 的核心就是让 Git 提交信息具备统一格式：

最常用的类型是：

  • feat：新功能；
  • fix：修复 Bug；
  • docs：文档；
  • style：代码格式；
  • refactor：重构；
  • test：测试；
  • perf：性能优化；
  • build：构建系统；
  • ci：CI/CD；
  • chore：工程维护。

规范化提交信息的好处是：

  • 提交历史更清晰；
  • 代码回溯更方便；
  • 版本发布更容易整理 changelog；
  • 多人协作时更容易理解每次提交的目的。
