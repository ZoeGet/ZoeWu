Codex更新桌面快捷方式PowerShell脚本

Created2026-07-09|Updated2026-07-09
|Post Views:

问题背景

从 Microsoft Store 安装的 Codex 是 AppX 包应用，和传统的 exe 安装程序不一样。如果你不小心删了桌面的快捷方式，想再建一个，常规的”发送到桌面快捷方式”对这类应用并不好使——右键菜单里根本没有这个选项。

更麻烦的是，如果用传统的 New-Object -ComObject WScript.Shell 手动创建 .lnk，指向的是 shell:AppsFolder\...，虽然能用，但快捷方式的图标可能不是 Codex 的官方图标，看起来总不太对劲。

脚本思路

这个脚本通过 Shell.Application COM 对象枚举 shell:AppsFolder 中的应用列表，找到 Codex，然后调用它的 Verbs().DoIt() 来触发 Windows 原生的”创建快捷方式”动作——和你在右键菜单里点”创建快捷方式”是同一个操作，因此生成的文件一定会带上 Codex 的官方图标。

额外做了几件事：

  • 自动识别已有的官方图标快捷方式——如果桌面上已经有了，直接提示并退出，不会重复创建。
  • 容错处理——如果创建后几秒内没有检测到新文件，会输出提示，不盲目报错。
  • 兼容性——兼容中文/英文系统语言（创建快捷方式 / Create shortcut / shortcut）。

使用方法

  1. 将以上脚本另存为 New-CodexShortcut.ps1。
  2. 右键 → 使用 PowerShell 运行（或以管理员身份运行，视系统权限而定）。
  3. 脚本会自动检测桌面状态，如果已有官方快捷方式则直接退出，否则调用系统接口创建一个新的。

注： 如果系统里 Codex 的应用显示名称不是 Codex（比如在某些区域版本下），可能需要将第 7 行的 -eq 'Codex' 改为对应的显示名称。

脚本原理（摘要）

步骤 说明
枚举应用 通过 Shell.Application 访问 shell:AppsFolder
匹配 Codex 按名称过滤，取第一个匹配项
解析 AppUserModelID 用于后续识别官方快捷方式的关键字段
触发系统动作 调用 Verbs().DoIt() 模拟右键菜单的”创建快捷方式”
轮询检测 创建后 10 秒内持续扫描桌面，确认生成结果
