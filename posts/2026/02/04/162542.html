安装步骤

全局安装OpenSpec/升级OpenSpec

npm install -g @fission-ai/openspec@latest

在项目中初始化OpenSpec

cd your-project
openspec init
  • 初始化过程会：
    • 1.提示选择原生支持的AI工具
    • 2.自动配置所选工具的斜杠命令
    • 3.创建openspec/目录结构
    • 4.在项目根目录写入管理的AGENTS.md存根

验证设置

# 验证设置并查看活动变更
    openspec list

# 检查工具集成状态
    openspec doctor

刷新AI指导

​ 在每个项目中运行此程序，以重新生成AI指导，并确保最新的斜杠命令处于激活状态：

openspec update

目录结构

openspec/
├── specs/
│   └── auth/
│       └── spec.md             # 当前认证规范（如果存在）
└── changes/
    └── add-2fa/                # AI创建的完整结构
        ├── proposal.md         # 为什么和什么变更
        ├── tasks.md            # 实施检查清单
        ├── design.md           # 技术决策（可选）
        └── specs/
            └── auth/
                └── spec.md     # 显示增量的差异

部署过程

1.在VSCode的终端命令行输入命令安装OpenSpec，并使用openspec init创建一个新的 openspec/ 目录结构

2.OpenSpec 支持多种开发工具，我这里使用Cursor

3.选择完成后按Tab进行保存

总结

OpenSpec 将规范驱动开发与 AI 编码助手完美结合：

  1. AI 更懂你的项目：规范让 AI 理解项目架构和约定
  2. 代码质量更高：统一的规范确保代码一致性
  3. 文档永不过时：规范随代码演进，形成活文档
  4. 团队协作更顺畅：共享规范，统一理解
  5. 知识有效沉淀：完整的变更历史和决策记录

在 Cursor 中使用 OpenSpec，你可以充分发挥 AI 的能力，同时保持对代码质量的控制。

实践建议

开始使用 OpenSpec

  1. 从小做起：选择一个小功能开始
  2. 保持简单：不要一开始就追求完美的规范
  3. 持续迭代：根据团队反馈不断优化
  4. 分享经验：在团队中分享成功案例

编写好规范的技巧

  1. 使用 GIVEN-WHEN-THEN 格式：清晰描述场景
  2. 关注行为而非实现：描述“做什么”而非“怎么做”
  3. 包含边界情况：考虑异常和特殊情况
  4. 保持可验证性：每个需求都应该可以测试
  5. 使用示例：提供具体的输入输出示例

Ensp模拟器中交换机无法启动，命令行界面只显示The device is running的解决办法。
我的电脑在删除D盘（ensp原来的安装位置），并把D盘的所有软件移动到E盘后，重装ensp便出现了这个问题。路由器、防火墙等均未受影响，只有交换机在启动时，有的能启动，有的一直卡在The device is running，不显示命令行。 在网上寻找了许多方法，类似于关闭防火墙，cmd中输入lodctr /r的命令，检查虚拟网卡是否为192.168.56.0/24等均未解决。 偶然看到一篇文章，内容中提到关闭Hyper-V，突然想到自己之前似乎开启过这个功能。 cmd切换到管理员模式输入： 1bcdedit /set hypervisorlaunchtype off 经测试问题已解决。
